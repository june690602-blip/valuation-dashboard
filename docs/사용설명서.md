# 투자지표 대시보드 사용설명서

> 처음 쓰는 분은 2~3장의 빠른 사용법부터, 계산 근거가 필요한 분은 4장 이후의 상세 방법론을 보세요.
> 앱 안의 **사용설명서**는 실제 화면 옆에 띄워 따라 하기 좋은 축약판이고, 이 문서는 계산 방식과 한계까지 담은 상세판입니다.

---

## 0. 이 프로그램이 하는 일 한 줄 요약

**종목 하나를 넣으면, "지금 이 주가가 적정한가? 아니라면 왜인가?"** 를
① 업종 비교 ② 자기 과거 ③ 이론가치(RIM) 세 각도에서 계산하고,
베타·WACC 같은 자본비용까지 붙여 **직관적인 판정과 근거 문장**으로 보여줍니다.

엄밀한 무위험 차익거래(arbitrage)가 아니라, **가치와 가격의 괴리(mispricing)를 찾는
스크리닝 보조 도구**입니다.

---

## 1. 설치와 실행

```bash
pip install -r requirements.txt      # 최초 1회
streamlit run app.py                 # 실행 (브라우저가 자동으로 열림)
```

- 인터넷만 되면 **핵심 분석은 API 키·로그인 없이** 동작합니다. Gemini와 OpenDART 키는 일부 고급 기능의 선택 사항입니다.
- **첫 조회는 수십 초** 걸릴 수 있습니다(피어 10여 개의 데이터를 모으기 때문). 이후는 캐시되어 빠릅니다.
- 데이터가 이상하면 사이드바 **"데이터 캐시 비우기"** 를 누르고 다시 조회하세요.

## 2. 처음 5분: 화면을 읽는 순서

1. **분석 대상을 고릅니다.** 주식·채권·포트폴리오 중 지금 답을 얻고 싶은 도구를 엽니다.
2. **입력값을 확인합니다.** 주식은 시장과 종목, 채권은 국가·만기, 포트폴리오는 자산과 비중을 먼저 맞춥니다.
3. **결론부터 읽습니다.** 판정이나 핵심 지표를 본 뒤 그 결론을 만든 근거 화면으로 내려갑니다.
4. **반대 근거를 찾습니다.** 신뢰도·표본 수·현금흐름·공시와 뉴스가 결론을 흔들지 않는지 확인합니다.

주식 가치평가의 가정은 사이드바에서 조정합니다.

- **무위험이자율 R_f**: CAPM·WACC의 출발점이며 채권 화면의 10년물 국채 금리를 기본값으로 씁니다.
- **시장위험프리미엄 MRP**: 주식시장이 무위험자산보다 추가로 요구하는 수익률입니다.
- **피어 수**: 업종 비교에 사용할 경쟁사 수(5~15)입니다.
- **RIM 요구수익률 직접 지정**: 켜면 RIM의 요구수익률을 직접 입력합니다. 기본값은 CAPM 자기자본비용(k_e)입니다.

가정을 바꾸면 WACC와 적정주가가 다시 계산됩니다. 한 가정이 결론을 뒤집는지 살펴보는 **민감도 분석**으로 활용하세요.

## 3. 주식 가치평가: 9개 화면 지도

권장 순서는 **결론 → 근거 → 반대 근거**입니다. 요약·판정에서 적정가 범위, 괴리율, 신뢰도를 먼저 본 뒤
밸류에이션·재무·업종 비교로 근거를 확인하고, 자본비용·백테스트·공시와 뉴스로 반대 근거를 찾으세요.

| 알고 싶은 것 | 화면 | 먼저 볼 것 |
|---|---|---|
| 이 회사는 무엇을 하나? | ① 기업·뉴스 | 사업 개요와 최근 이슈 |
| 결론은 무엇인가? | ② 요약·판정 | 적정가 범위·괴리율·신뢰도 |
| 가격 흐름은 어떤가? | ③ 주가차트 | 기간별 추세·지수 대비 성과 |
| 업종·과거 대비 싼가? | ④ 밸류에이션 | 업종·자기 역사·RIM의 방향 |
| 회사 체력은 괜찮은가? | ⑤ 재무 분석 | 매출·마진·현금흐름 |
| 경쟁사 중 어디쯤인가? | ⑥ 업종 비교 | 가치와 수익성의 상대 위치 |
| 자본비용보다 더 버는가? | ⑦ 자본비용 | ROIC와 WACC의 차이 |
| 역사적 밴드 신호가 과거에 통했나? | ⑧ 백테스트 | 표본 수와 이후 수익 분포 |
| AI가 계산 결과를 어떻게 해석하나? | ⑨ 종합 평가 | 스탠스·가격선·편입비중과 그 근거 |

> **신뢰도**는 여러 평가 방법이 얼마나 비슷한 값을 냈는지를 나타냅니다. 데이터 정확도나 수익 가능성을
> 보증하는 점수가 아닙니다. **백테스트**도 과거 관찰 결과일 뿐 미래 성과를 보증하지 않습니다.

## 4. 자본비용 탭 자세히 — 베타에서 WACC까지

CPA 1차에서 배운 CAPM/WACC를 이 종목의 **실제 과거 데이터로** 계산합니다.

**(1) 회귀 베타 β_L (레버드)**
최근 5년(최대 260주) **주간 수익률**을 시장지수(KOSPI/KOSDAQ/S&P500)에 회귀한
**기울기**가 β_L입니다. 함께 나오는 **R²** 는 설명력(0~1). R²가 낮으면(<0.1) "시장으로 설명 안 되는
개별 요인이 큰 종목"이니 베타를 덜 믿어야 합니다.

**(2) 무부채 베타 β_U (하마다 언레버링)**
```
β_U = β_L / (1 + (1 − t)·D/E)
```
β_L에는 **영업위험 + 재무위험(빚)** 이 섞여 있습니다. 하마다 식으로 부채 효과를 벗겨내면
**순수 영업위험만의 베타 β_U** 가 남습니다. (t=유효세율, D=이자부채, E=시가총액)

**(3) 자본비용 3형제**
```
k_U = R_f + β_U · MRP     ← 영업위험만 반영 (빚이 전혀 없다면 이만큼만 요구)
k_e = R_f + β_L · MRP     ← 재무위험까지 포함한 자기자본비용
재무위험 프리미엄 = k_e − k_U   ← 빚 때문에 주주가 추가로 요구하는 몫
```
이 분해가 이 도구의 특징입니다. "이 회사 주주 요구수익률 중 얼마가 **사업 위험**이고
얼마가 **차입 위험**인지"를 눈으로 보여줍니다.

**(4) 타인자본비용 k_d 와 WACC**
```
k_d = 이자비용 / 평균 이자부차입금        (재무제표 기반, 없으면 R_f+2%p로 가정)
WACC = k_e·E/(D+E) + k_d·(1 − t)·D/(D+E)
```
WACC 워터폴 차트가 자기자본·타인자본 기여를 쌓아 보여줍니다.

**(5) ROIC vs WACC — 가치를 창출하는가**
```
ROIC = 세후영업이익 / 투하자본(자기자본 + 이자부채 − 현금)
스프레드 = ROIC − WACC
```
- **ROIC > WACC** : 자본비용보다 더 번다 → **성장할수록 가치 창출** (좋음)
- **ROIC < WACC** : 자본비용도 못 번다 → **성장할수록 가치 잠식** (나쁨)

> 💡 금융업(은행·보험)은 부채가 곧 영업이라 하마다·WACC가 의미 없어 **자동으로 가려집니다**.
> 이때는 k_e(자기자본비용)와 PBR·ROE만 참고하세요.

## 5. 적정주가 3가지 방법 (판정의 근거)

| 방법 | 아이디어 | 공식(요지) | 잘 맞는 경우 |
|---|---|---|---|
| **① 업종 상대가치** | 비슷한 회사들은 비슷한 배수 | 피어 중앙값 PER×EPS, PBR×BPS, EV/EBITDA | 피어가 동질적일 때 |
| **② 역사적 밴드** | 자기 배수는 평균으로 회귀 | 5년 PER/PBR 25~75분위 × 현재 펀더멘털 | 사업이 안정적일 때 |
| **③ RIM(잔여이익)** | ROE가 요구수익률을 넘는 만큼 프리미엄 | `V = B + B(ROE−r)·w / (1+r−w)` | 흑자·자본이 정상일 때 |

- 사용 가능한 방법들의 평균으로 **괴리율** = 적정가 ÷ 현재가 − 1 을 구해 **5단계 배지**를 답니다.
  (+30%↑ 크게 저평가 / +10%~ 저평가 / ±10% 적정 / −10%~ 고평가 / −30%↓ 크게 고평가)
- **세 방법이 서로 많이 다르면 신뢰도 '낮음'**. 이땐 한 방법(예: 역사적 밴드)만 튀는 건 아닌지
  각 탭에서 확인하세요.
- **적자 기업**은 PER·RIM을 건너뛰고 PSR·PBR로, **자사주를 많이 산 기업**(장부자본이 비정상적으로 작음)은
  RIM을 건너뜁니다. 화면에 그 이유가 안내됩니다.

## 6. 자주 나오는 질문

**Q. R_f(무위험이자율)는 뭘 기준으로 정했고, 왜 내가 바꾸게 해놨나요?**
- Streamlit 화면은 **현재 10년 만기 국채 수익률**을 우선 사용하고 불러오지 못하면 정적 가정치로 대체합니다.
  별도 정적 웹 화면은 현재 KR 3.5% / US 4.5%의 사전 설정값을 사용할 수 있으므로, 화면에 표시된 값과 출처를 기준으로 해석하세요.
- 슬라이더로 열어둔 이유:
  1. **정답이 하나가 아니다.** 실무는 10년 국채를 많이 쓰지만, 평가 기간·유파에 따라 3년물·기준금리를
     쓰기도 합니다. 어떤 값을 썼는지 **투명하게 드러내는 게** 숨기는 것보다 낫습니다.
  2. **금리는 변한다.** 하드코딩하면 금방 낡습니다.
  3. **민감도 분석.** WACC·RIM 적정가는 R_f에 민감합니다. 슬라이더로 3%→5% 바꿔 보며
     **"금리 가정이 판정을 뒤집는가"** 를 직접 확인하는 것이 오히려 바람직합니다.

**Q. 요약 지표의 PER와 밸류에이션 화면의 '외부 제공 참고치' PER가 왜 다른가요?**
- 요약/계산값은 **재무제표 기반 TTM(최근 4분기 합산)** 으로 이 프로그램이 직접 계산합니다.
- 외부 제공 참고치는 네이버·야후가 게시한 값으로, **EPS 기준(연결/별도, 지배지분 여부)·기준시점·주식수**
  처리가 달라 차이가 납니다. 어느 쪽이 틀린 게 아니라 **정의가 다른 것**입니다. 표에 둘 다 보여주는 이유입니다.
- 한국 종목은 **OpenDART 공시 원본**(연결, ~6개년)을 우선 사용하도록 개선해, 연간 기준
  ROE·PBR·역사적 밴드·백테스트를 공시 원본 기반 수치에 맞춥니다(헤더에 `📄 재무 DART 연결` 표시).
  다만 헤더의 **PER/ROE(TTM)** 는 최신성을 위해 아직 yfinance 분기 합산이라, 네이버 트레일링 값과
  소폭 다를 수 있습니다(완전 일치는 DART 분기 연동 시 — 다음 단계).

**Q. 이걸로 바로 매매해도 되나요?**
- **아니요.** 무료 데이터라 결측·오차가 있고, 단일 시점의 밸류에이션은 촉매·업황·지배구조 리스크를
  담지 못합니다. **가설을 세우고 걸러내는 1차 스크리너**로 쓰고, 실제 판단은 공시 원문과 함께 하세요.

## 7. 백테스트 탭 — 신뢰도 점검

"쌀 때 샀으면 정말 수익이 좋았나?"를 **이 종목의 과거로** 검증합니다.

- **검증 대상**: **역사적 밴드** 신호 하나(자기 PER/PBR 백분위). 종합 판정 전체가 아닙니다
  (과거 시점의 피어·재무를 되살리면 룩어헤드/생존편향이 생겨 제외).
- **룩어헤드 방지**: '그날 쌌는지'는 **그날까지의 과거 분포**(롤링 백분위)로만 판단합니다.
- **읽는 법**:
  - **순위상관**이 양수(+)면 "저평가율이 클수록 이후 수익↑" = 평균회귀 신호가 과거에 유효했다는 뜻.
  - **구간별 막대**: 저평가 구간의 이후 수익이 고평가 구간보다 높으면 신호가 유효.
  - **누적수익 곡선**: '저평가일 때만 보유' 예시 전략 vs 단순 보유 vs 지수. **거래비용·세금 미반영**.
- **한계(꼭 기억)**: 단일 종목·짧은 표본(연간 재무가 4~5년뿐)이라 **과최적화·생존편향**에 취약합니다.
  삼성전자처럼 최근 내내 비쌌던 종목은 **'저평가 구간' 표본이 0** 일 수 있습니다. **경향만** 참고하세요.

## 8. 한계와 주의 (반드시 읽기)

- 데이터 출처: 한국 연간 재무 **OpenDART 공시 원본**(우선), 재무 보완·시세 yfinance,
  한국 참고 멀티플 네이버금융, 시총·업종 FinanceDataReader(KRX). **무료 소스라 항목 결측·오차·지연**이 있습니다.
- OpenDART 키가 없으면 자동으로 yfinance 재무로 폴백합니다(앱은 그대로 동작, 재무 이력만 짧아짐).
- KRX **업종분류가 실제 사업과 다를 수 있습니다**(예: 삼성전자 = '통신 및 방송 장비 제조업' → 피어가 어색).
- 베타는 기간·주기(주간)에 따라 야후/네이버 표기와 **±0.2 정도 차이**는 정상입니다.
- 이 도구는 **투자 조언이 아니라 학습·분석 보조**입니다. 최종 판단과 책임은 이용자에게 있습니다.

## 부록. 용어 빠른 정리

| 약어 | 뜻 | 이 도구에서 |
|---|---|---|
| TTM | Trailing Twelve Months | 최근 4개 분기 합산(최신 실적 반영) |
| RIM | Residual Income Model, 잔여이익모델 | ROE가 r을 넘는 초과이익으로 적정 PBR 산출 |
| β_L / β_U | 레버드/무부채 베타 | 하마다 식으로 상호 변환 |
| k_e / k_d | 자기자본/타인자본 비용 | CAPM·이자비용 기반 |
| WACC | 가중평균자본비용 | 할인율·요구수익률의 종합 |
| ROIC | 투하자본이익률 | WACC와 비교해 가치창출 판정 |
| 밸류트랩 | 싸 보이지만 이유 있는 저평가 | 역성장·수익성 훼손이 낮은 배수를 정당화 |

## 부록 2. 선택 기능: Gemini

가치평가·재무·차트·채권·포트폴리오와 규칙 기반 해설은 Gemini 키 없이도 사용할 수 있습니다.
키를 연결하면 Gemini가 업종·비교기업 선정, 기업 소개 번역, 뉴스 분류·요약, 종합 평가에 참여할 수 있습니다.

[Google AI Studio](https://aistudio.google.com)에서 API 키를 발급받아 `.streamlit/secrets.toml`의
`GEMINI_API_KEY = "..."`에 넣거나 환경변수로 설정한 뒤 앱을 다시 실행하세요. 키를 코드나 저장소에
커밋하지 마세요.

> 종합 평가는 매수·회피 같은 스탠스, 목표가·손절선, 성향별 편입비중까지 생성할 수 있습니다.
> 이는 검증된 투자 조언이 아닌 언어모델 출력입니다. 실제 주문 기준으로 사용하지 말고 수치와 출처를
> 공시·뉴스 원문으로 교차 확인하세요.
